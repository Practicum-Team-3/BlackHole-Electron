Creating a new Scenario

To create a new scenario you must already be connected to the tool. This process will allow you to add systems to a uniquely named scenario. You may also edit the scenario after creating it or delete it. You have to create a scenario to run attacks from one system to another.

Example: To create a new Scenario

  1. Click to the first tab of the “Overviews Panel” (the panel on the left). This is the Scenarios tab.
  2. Click on the button labeled “Create Scenario”, located on the bottom of the panel. A dialog box will appear.
  3. Enter a unique name on the first text box.
  4. Enter a brief description of the scenario on the second text box (Optional).
  5. Click on the button labeled “Create”.

NOTE: The tool only allows you to enter only a unique name of each scenario, therefore, names may not be repeated.

Delete a Scenario

To delete a scenario you must already be connected to the tool. To delete a scenario, the scenario must have already been created. This process will allow you to delete a scenario from the tool without deleting a duplicate system from another scenario.

Example: To delete a new Scenario

  1. Click to the first tab of the “Overviews Panel” (the panel on the left). This is the Scenarios tab.
  2. Click on the name of the Scenario you want to delete.
  3. Click on the red trash bin button. A dialogue box should appear.
  4. Click Yes to confirm you want to delete the Scenario.

Save a Scenario

To save changes made to a scenario you must be connected to the tool. To save a scenario, the scenario must have already been created. This process will allow you to save any modifications made to a scenario.

Example: To save a Scenario

  1. Click to the first tab of the “Overviews Panel” (the panel on the left). This is the Scenarios tab.
  2. Click on the name of the Scenario you want to save.
  3. Click on the green button under the Scenario to open the scenario onto the "Network Graph".
  4. Click the save button, located at the top left corner of the screen. The save button will be an icon of a floppy disk. You should see a notification box appear on the top right corner once the scenario has been successfully saved.

Run a Scenario

To run a scenario, you must be connected to the tool. To run a scenario, the scenario must already have been created. This process will allow you to run attacks from boxes that have already been set up with exploits and programs.

Example: To run a Scenario

  1. Click to the first tab of the “Overviews Panel” (the panel on the left). This is the Scenarios tab.
  2. Click on the name of the Scenario you want to save.
  3. Click on the green button under the Scenario to open the scenario onto the "Network Graph".
  4. Click the run button, located second to the top left corner of the screen. The run button will be an icon of a play button. You should see a notification box appear on the top right corner with a progress bar of the scenario running. Once the progress bar is complete, the same notification window will let you know the scenario has run successfully.

Add a Box from Vagrant

This process will allow you to add boxes to the system from Vagrant to be able to run a scenario and attacks from one system to another.

Example: To Add a Box from Vagrant

  1. Click the second tab of the "Overviews Panel"(the panel on the left). This is the Boxes tab.
  2. Click the bottom right button labeled, "Add from vagrant". A dialogue window should appear.
  3. Enter the ID name for the box you want to add from Vagrant.
  4. Click on the button labeled, "Download".

Add a Box to a Scenario

To add a Box to a Scenario, the scenario must have already been created. This process will allow you to add boxes to a scenario to run attacks from one system to another.

Example: To add a Box to a Scenario

  1. Click to the first tab of the “Overviews Panel” (the panel on the left). This is the Scenarios tab.
  2. Click on the name of the Scenario you want to save.
  3. Click on the green button under the Scenario to open the scenario onto the "Network Graph".
  4. Click the second tab of the "Overviews Panel"(the panel on the left). This is the Boxes tab.
  5. Click the name of the Box you want to add to the scenario.
  6. Click click on the green button, in green and labeled as "+", located under the name of the Box.

Remove a Box from a Scenario

To remove a box from a scenario, the box must have already been uploaded into the tool and added to a scenario. This process will allow you to remove a box from the tool without removing the box from other scenarios or the entire tool.

Example: To Remove a Box from a Scenario

  1. Click to the first tab of the “Overviews Panel” (the panel on the left). This is the Scenarios tab.
  2. Click on the name of the Scenario you want to remove a Box from.
  3. Click on the green button under the Scenario to open the scenario onto the "Network Graph".
  4. Click on the node on the Network Graph that represents the box you want to remove.
  5. On the right side panel, the "Settings Panel", click the button labeled as "Removed" and in red.

Delete a Box from the system

To delete a box, the box must have already been uploaded into the tool. This process will allow you to delete a box from the tool.

Example: To Delete a Box from the tool

  1. Click to the second tab of the “Overviews Panel” (the panel on the left). This is the Boxes tab.
  2. Click on the name of the Box you want to delete from the tool.
  3. Click on the red button under the Box name labeled with a "trash bin" icon.
  4. Click on the button labeled, "Yes" to confirm deletion.

Box Settings

This document makes the assumption that you are knowledgeable in the purpose of an automated malware and exploit data collection tool. You should have some knowledge in basic system settings, such as motherboard memory and processors. Lastly, you should have experience in operating buttons, drop down menus, and drag and drop on windows.

Example: To change the Settings of a Box

  1. Click to the first tab of the “Overviews Panel” (the panel on the left). This is the Scenarios tab.
  2. Click on the name of the Scenario you want to change the setting of a Box from.
  3. Click on the green button under the Scenario to open the scenario onto the "Network Graph".
  4. Click on the node on the Network Graph that represents the box you want to change the settings of.
  5. On the right side panel, the "Settings Panel", you can make any changes using the drop-down menus and sliders.

    6. NOTE: Changes in a Box will be saved after you have saved the scenario or have run the scenario.

View different Scenarios

Through this process you will be able to view different scenarios.

Example: To view a different scenario

  1. Click to the first tab of the “Overviews Panel” (the panel on the left). This is the Scenarios tab.
  2. Click on the name of the Scenario you want to view.
  3. Click on the green button under the Scenario to open the scenario onto the "Network Graph".
  4. Above the "Network Graph" you can click on the tab of the scenario you want to view/make changes to.

Upload an Exploit

This process will allow you to upload Exploits from your directory to the system. These exploits can then be installed in a box, used to attack other systems.

Example: To Upload an Exploit to the system

  1. Click to the third tab of the “Overviews Panel” (the panel on the left). This is the Exploits tab.
  2. Click on the button labeled, "Upload Exploit", located on the bottom right of the screen. A dialogue window should open.
  3. Click on the "browse" button, to select the file from your computer's directory, and choose the file.
  4. Select from the drop-down menu the target OS.
  5. Enter a brief description of the exploit.
  6. Click on the button labeled, "Upload".

Add an Exploit

To add an exploit to a box, the exploit must already be uploaded to the system. This process will allow you to automatically install an exploit to a Box.

Example: To add an Exploit to a Box

  1. Click to the third tab of the “Overviews Panel” (the panel on the left). This is the Exploits tab.
  2. Click on the name of the Exploit you want to add to a Box.
  3. Click on the green button labeled, "+". A dialogue box should appear.
  4. Enter the path of the box the exploit should be installed to.
  5. Click on the button labeled, "Save".

    6. NOTE: The exploit will be installed into the box until the scenario is ran.

Delete an Exploit

This process allow you to Delete an Exploit from the system.

Example: To Delete an Exploit from the system

  1. Click to the third tab of the “Overviews Panel” (the panel on the left). This is the Exploits tab.
  2. Click on the the name of the exploit you would like to delete.
  3. Click on the red button labeled with a "trash bin" icon, located under the name of the Exploit. A dialogue box should appear.
  4. Click on the button labeled "Yes" in the dialogue box, to confirm the deletion.

RDP

This process will allow you to RDP from any box and any scenario. A scenario and a box must have already been created.

Example: RDP to a box

  1. Click to the first tab of the “Overviews Panel” (the panel on the left). This is the Scenarios tab.
  2. Click on the the name of the Scenario you would like to choose a Box from to RDP.
  3. Click on the node in the Network Graph that represents the Box you want to RDP.
  4. Click on the third icon from the top left of the screen, labeled "RDP". A screen should appear with the RDP.
